# Fase 56 — Limites de carteira por corretor

O gerente configura capacidade total, capacidade por projeto, percentual de alerta e motivo. Os limites representam disponibilidade operacional; não são meta, score ou ranking de pessoas.

O bloqueio vive no banco e alcança cadastro com responsável, distribuição, transferência individual, transferência entre equipes e cobertura de ausência. Uma trava por corretor evita ultrapassagem concorrente. Leads ganhas, perdidas, descartadas e arquivadas não consomem capacidade. Reduzir o teto não remove a carteira atual, apenas impede novas entradas até a normalização.

Homologar gerente, superintendente, diretor, dois tenants, corretor de outra equipe, limites inválidos, alerta, carga no teto, projetos distintos, atribuições simultâneas e todos os caminhos de transferência. Execute `npm run broker-capacity:check`.
